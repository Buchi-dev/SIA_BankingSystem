Wired City Banking System API Docs

A modern, user-friendly API documentation for your city banking project. Use the sidebar to navigate between sections.

Overview

This documentation provides an overview of the server-side architecture, including controllers, models, routes, and features.

Project Purpose: Placeholder for a brief description of the Wired City Banking System and its goals.

Features

  • User registration, authentication, and profile management
  • Account creation, viewing, updating, deletion, and balance checking
  • Transaction operations: deposit, withdraw, transfer, and history
  • External system registration, payment, and balance inquiry
  • RESTful API with Express.js and MongoDB (Mongoose)
  • CORS and JSON body parsing middleware

More features coming soon...

Models

User

  • username (unique, required)
  • password_hash (required)
  • role (customer/admin)
  • created_at
{
  "username": "johndoe",
  "password_hash": "...",
  "role": "customer",
  "created_at": "2025-06-17T12:00:00Z"
}

Account

  • user_id (ref: User, required)
  • balance (default: 0)
  • status (active/inactive/closed)
  • created_at
{
  "user_id": "60f7c2...",
  "balance": 1000,
  "status": "active",
  "created_at": "2025-06-17T12:00:00Z"
}

Transaction

  • from_account_id, to_account_id (ref: Account)
  • amount
  • type (deposit/withdrawal/transfer)
  • timestamp
  • status (pending/completed/failed)
{
  "from_account_id": "60f7c2...",
  "to_account_id": "60f7c3...",
  "amount": 500,
  "type": "transfer",
  "timestamp": "2025-06-17T12:00:00Z",
  "status": "completed"
}

ExternalSystem

  • system_name (unique, required)
  • api_key (required)
  • contact
  • registered_at
{
  "system_name": "CityParking",
  "api_key": "abc123...",
  "contact": "admin@cityparking.com",
  "registered_at": "2025-06-17T12:00:00Z"
}

Controllers

User Controller

  • register: Register a new user
  • login: Authenticate user credentials
  • profile: Get user profile (excluding password)

Account Controller

  • create: Create a new account
  • view: View account details
  • update: Update account info
  • delete: Delete an account
  • balance: Check account balance

Transaction Controller

  • deposit: Deposit funds
  • withdraw: Withdraw funds
  • transfer: Transfer funds between accounts
  • history: Get transaction history

External System Controller

  • register: Register an external system
  • payment: Process a payment (placeholder)
  • balanceInquiry: External system balance inquiry (placeholder)

Routes

User Routes (/api/users)

  • POST /register
    Register a new user.
    Request Example: 
    {
  "username": "johndoe",
  "password": "password123"
}
    Response Example: 
    {
  "message": "User registered successfully"
}
  • POST /login
    Authenticate user credentials.
    Request Example: 
    {
  "username": "johndoe",
  "password": "password123"
}
    Response Example: 
    {
  "token": "jwt-token-here"
}
  • GET /profile
    Get user profile (requires authentication).
    Response Example: 
    {
  "username": "johndoe",
  "role": "customer"
}

Account Routes (/api/accounts)

  • POST /create
    Create a new account for the authenticated user.
    Response Example: 
    {
  "account_id": "60f7c2...",
  "balance": 0,
  "status": "active"
}
  • GET /view/:id
    View account details by account ID.
    Response Example: 
    {
  "account_id": "60f7c2...",
  "user_id": "60f7c1...",
  "balance": 1000,
  "status": "active"
}
  • PUT /update/:id
    Update account information.
    Request Example: 
    {
  "status": "inactive"
}
    Response Example: 
    {
  "message": "Account updated successfully"
}
  • DELETE /delete/:id
    Delete an account by ID.
    Response Example: 
    {
  "message": "Account deleted successfully"
}
  • GET /balance/:id
    Check account balance by account ID.
    Response Example: 
    {
  "balance": 1000
}

Transaction Routes (/api/transactions)

  • POST /deposit
    Deposit funds into an account.
    Request Example: 
    {
  "account_id": "60f7c2...",
  "amount": 500
}
    Response Example: 
    {
  "message": "Deposit successful",
  "balance": 1500
}
  • POST /withdraw
    Withdraw funds from an account.
    Request Example: 
    {
  "account_id": "60f7c2...",
  "amount": 200
}
    Response Example: 
    {
  "message": "Withdrawal successful",
  "balance": 1300
}
  • POST /transfer
    Transfer funds between accounts.
    Request Example: 
    {
  "from_account_id": "60f7c2...",
  "to_account_id": "60f7c3...",
  "amount": 100
}
    Response Example: 
    {
  "message": "Transfer successful",
  "from_balance": 1200,
  "to_balance": 1100
}
  • GET /history
    Get transaction history for an account.
    Response Example: 
    [
  {
    "type": "deposit",
    "amount": 500,
    "timestamp": "2025-06-17T12:00:00Z"
  },
  {
    "type": "withdrawal",
    "amount": 200,
    "timestamp": "2025-06-17T13:00:00Z"
  }
]

External System Routes (/api/external-systems)

  • POST /register
    Register an external system.
    Request Example: 
    {
  "system_name": "CityParking",
  "api_key": "abc123...",
  "contact": "admin@cityparking.com"
}
    Response Example: 
    {
  "message": "External system registered successfully"
}
  • POST /payment
    Process a payment (placeholder).
    Request Example: 
    {
  "system_name": "CityParking",
  "amount": 100
}
    Response Example: 
    {
  "message": "Payment processed"
}
  • GET /balance-inquiry
    External system balance inquiry (placeholder).
    Response Example: 
    {
  "system_name": "CityParking",
  "balance": 5000
}
